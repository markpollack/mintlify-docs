## Overview

Agent Experiment is a **purpose-built Java experiment driver** for evaluating AI coding agents. It provides a complete pipeline for running reproducible experiments: load fixture datasets, invoke agents, judge results with [Agent Judge](/projects/incubating/agent-judge) juries, persist structured results, and compare runs across variants.

The framework is agent-agnostic at its core. The `experiment-core` module has no AI SDK dependencies, while `experiment-claude` adds Claude Code SDK integration for agent invocation, LLM-based planning, and semantic evaluation.

## Quick Start

### Maven Dependency

```xml
<dependency>
    <groupId>io.github.markpollack</groupId>
    <artifactId>experiment-core</artifactId>
    <version>0.1.0-SNAPSHOT</version>
</dependency>

<!-- For Claude integration -->
<dependency>
    <groupId>io.github.markpollack</groupId>
    <artifactId>experiment-claude</artifactId>
    <version>0.1.0-SNAPSHOT</version>
</dependency>
```

Add the snapshot repository:

```xml
<repositories>
    <repository>
        <id>central-snapshots</id>
        <url>https://central.sonatype.com/repository/maven-snapshots/</url>
        <snapshots><enabled>true</enabled></snapshots>
        <releases><enabled>false</enabled></releases>
    </repository>
</repositories>
```

## Running an Experiment

### 1. Configure

```java
ExperimentConfig config = ExperimentConfig.builder()
    .experimentName("my-experiment")
    .datasetDir(Path.of("datasets/my-benchmark"))
    .model("sonnet")
    .promptTemplate("Your task: {{task}}")
    .perItemTimeout(Duration.ofMinutes(10))
    .outputDir(Path.of("results"))
    .build();
```

### 2. Wire Dependencies

```java
DatasetManager datasetManager = new FileSystemDatasetManager();
ResultStore resultStore = new FileSystemResultStore(resultsDir);
Jury jury = SimpleJury.builder()
    .name("eval")
    .judge(myJudge)
    .votingStrategy(VotingStrategy.majority())
    .build();
```

### 3. Invoke

```java
AgentInvoker invoker = new ClaudeSdkInvoker(ClaudeSdkInvokerConfig.defaults());

ExperimentRunner runner = new ExperimentRunner(
    datasetManager, jury, resultStore, config);
ExperimentResult result = runner.run(invoker);

// Result contains per-item scores, costs, durations, and aggregate metrics
System.out.println("Pass rate: " + result.passRate());
System.out.println("Total cost: $" + result.totalCostUsd());
```

## Datasets

Datasets are Git-managed directories with a `dataset.json` manifest:

```
my-benchmark/
  dataset.json          # Manifest with items list
  items/
    item-1/
      item.json         # Item metadata (task, difficulty, tags)
      before/           # Starting state (optional)
      reference/        # Expected result (optional)
    item-2/
      ...
```

### Loading and Filtering

```java
DatasetManager dm = new FileSystemDatasetManager();
Dataset dataset = dm.load(Path.of("datasets/my-benchmark"));

// Filter by difficulty bucket
List<DatasetItem> easy = dm.filteredItems(dataset,
    ItemFilter.bucket("easy"));

// Filter by tags
List<DatasetItem> tagged = dm.filteredItems(dataset,
    ItemFilter.tags(Set.of("maven", "spring-boot")));

// Get current version (git SHA + dirty flag)
DatasetVersion version = dm.currentVersion(dataset);
```

### Resolving Items

```java
ResolvedItem resolved = datasetManager.resolve(item);

Path beforeDir = resolved.beforeDir();       // Starting state
Path referenceDir = resolved.referenceDir(); // Expected result
SourceRef beforeRef = resolved.beforeRef();  // Git commit reference
```

## Agent Invocation

The `AgentInvoker` interface abstracts agent execution:

```java
@FunctionalInterface
public interface AgentInvoker {
    InvocationResult invoke(InvocationContext context);
}
```

### InvocationContext

```java
InvocationContext context = InvocationContext.builder()
    .workspace(workspace)
    .prompt("Increase test coverage to 80%")
    .systemPrompt("You are a Java testing expert.")
    .model("sonnet")
    .timeoutHint(Duration.ofMinutes(10))
    .build();
```

### InvocationResult

```java
InvocationResult result = invoker.invoke(context);

result.success();           // Did the agent complete?
result.terminalStatus();    // COMPLETED, TIMEOUT, or ERROR
result.costUsd();           // API cost in USD
result.totalTokens();       // Total tokens consumed
result.duration();          // Wall-clock duration
result.sessionId();         // Agent session identifier
```

### Claude SDK Invoker

```java
ClaudeSdkInvokerConfig config = ClaudeSdkInvokerConfig.builder()
    .maxBudgetUsd(5.0)
    .maxTurns(50)
    .build();

AgentInvoker invoker = new ClaudeSdkInvoker(config);
```

## Pipeline (Analyze + Plan + Execute)

`PipelineAgentInvoker` wraps any `AgentInvoker` with optional analysis and planning phases:

```java
// Analyze project structure (deterministic, no AI calls)
ProjectAnalyzer analyzer = new MavenProjectAnalyzer();

// Generate execution plan (uses Claude)
PlanGenerator planner = new ClaudePlanGenerator(knowledgeDir);

PipelineConfig pipelineConfig = PipelineConfig.builder()
    .enableAnalysis(true)
    .enablePlanning(true)
    .knowledgeDir(Path.of("knowledge"))
    .build();

// Wraps the base invoker with analyze → plan → execute phases
AgentInvoker pipeline = new PipelineAgentInvoker(
    analyzer, planner, pipelineConfig, baseInvoker);
```

Analysis and planning failures gracefully degrade (log warning, skip phase) - only execution failures propagate.

## Comparing Experiments

```java
ComparisonEngine engine = new DefaultComparisonEngine(resultStore);

ComparisonResult comparison = engine.compare(currentResult, baselineResult);

// Per-scorer summary
for (ScoreComparison sc : comparison.scoreComparisons()) {
    System.out.printf("%s: %.2f → %.2f (delta: %+.2f)%n",
        sc.scorerName(), sc.baselineMean(), sc.currentMean(), sc.delta());
}

// Per-item diffs
for (ItemDiff diff : comparison.itemDiffs()) {
    System.out.printf("Item %s: %s%n", diff.itemId(), diff.status());
    // IMPROVED, REGRESSED, UNCHANGED, NEW, or REMOVED
}

// Detect regressions above threshold
List<ItemDiff> regressions = engine.detectRegressions(
    comparison, Map.of("build-judge", 0.1));
```

### Experiment Summary

```java
ExperimentSummary summary = engine.summarize(result);

summary.passRate();        // 0.0–1.0
summary.totalCostUsd();    // Aggregate cost
summary.totalTokens();     // Aggregate tokens
summary.scoreAggregates(); // Per-judge mean scores
```

## Sessions and Sweeps

Sessions group variant results within a single run. Sweeps group sessions across multiple runs.

### Sessions

```java
SessionStore sessionStore = new FileSystemSessionStore(outputDir);

// Create session
RunSession session = sessionStore.createSession(
    "run-1", "my-experiment", Map.of("git-sha", commitSha));

// Run each variant
for (String variant : List.of("control", "variant-a", "variant-b")) {
    ActiveSession active = new ActiveSession("run-1", "my-experiment", variant);
    ExperimentResult result = runner.run(invoker, active);
    sessionStore.saveVariantToSession("run-1", "my-experiment", variant, result);
}

// Finalize
sessionStore.finalizeSession("run-1", "my-experiment", RunSessionStatus.COMPLETED);
```

### Sweeps

```java
SweepStore sweepStore = new FileSystemSweepStore(outputDir);

// Create sweep with expected variants
Sweep sweep = sweepStore.createSweep("sweep-1", "my-experiment",
    List.of("control", "variant-a", "variant-b"), metadata);

// Add sessions as they complete
sweepStore.addSession("sweep-1", "my-experiment", "run-1", gitCommitSha);

// Check completeness
sweep = sweepStore.loadSweep("my-experiment", "sweep-1").get();
List<String> missing = sweep.missingVariants(); // Variants not yet resolved
boolean complete = sweep.isComplete();
```

## Result Storage

Results persist as JSON files on disk:

```java
ResultStore store = new FileSystemResultStore(Path.of("results"));

// Save
store.save(experimentResult);

// Query
Optional<ExperimentResult> loaded = store.load(experimentId);
List<ExperimentResult> history = store.listByName("my-experiment");
Optional<ExperimentResult> latest = store.mostRecent("my-experiment");
```

### ExperimentResult

```java
ExperimentResult result = runner.run(invoker);

result.experimentId();          // Unique run ID
result.experimentName();        // Configured experiment name
result.datasetVersion();        // Git SHA of dataset
result.passRate();              // Fraction of items that passed
result.totalCostUsd();          // Total API cost
result.totalTokens();           // Total tokens consumed
result.items();                 // List<ItemResult>
result.aggregateScores();       // Map<String, Double> per-judge means
result.knowledgeManifest();     // Snapshot of KB state (nullable)

// Per-item details
for (ItemResult item : result.items()) {
    item.itemId();
    item.passed();
    item.costUsd();
    item.scores();              // Map<String, Double> per-judge
    item.invocationResult();    // Full InvocationResult
    item.verdict();             // Agent Judge Verdict
}
```

## Diagnostics

Post-hoc analysis of experiment failures with gap classification:

```java
GapClassifier classifier = new HeuristicGapClassifier();
DiagnosticAnalyzer analyzer = new DiagnosticAnalyzer(classifier);

DiagnosticReport report = analyzer.analyze(result);

// Per-item diagnostics
for (ItemDiagnostic item : report.items()) {
    System.out.printf("Item %s: dominant gap = %s%n",
        item.itemId(), item.dominantGap());
}

// Aggregate distribution
GapDistribution dist = report.distribution();
System.out.println("Dominant gap category: " + dist.dominant());

// Actionable recommendations
for (String rec : report.recommendations()) {
    System.out.println("  → " + rec);
}
```

## Integration with Agent Judge

Agent Experiment uses [Agent Judge](/projects/incubating/agent-judge) for evaluation. Any `Jury` can be plugged into the `ExperimentRunner`:

```java
// Deterministic judges (agent-judge-core)
Judge buildJudge = BuildSuccessJudge.maven("compile");
Judge fileJudge = FileExistsJudge.of("target/classes/App.class");

// Compose into jury
Jury jury = SimpleJury.builder()
    .name("eval")
    .judge(buildJudge)
    .judge(fileJudge)
    .votingStrategy(VotingStrategy.majority())
    .build();

// Plug into runner
ExperimentRunner runner = new ExperimentRunner(
    datasetManager, jury, resultStore, config);
```

The `VerdictExtractor` utility normalizes jury verdicts into per-judge `[0, 1]` scores for aggregation in `ExperimentResult`.
